# docker搭建gitlab

GitHub 使用的是 Git 分布式版本控制系统。

GitLab 分为 社区版（Community Edition，缩写为 CE）和 企业版（Enterprise Edition，缩写为 EE）。社区版是免费的。本案例使用 gitlab-ce 社区版。

  • Docker搭建过程参考 服务器->常规部署
  • gitlab详细介绍请上网搜索

GitLab 的搭建有多种方式，本文案例使用 Docker 来搭建。

# 3.2 首次运行，初始化gitlab

  --restart always \
  -v /mnt/srv/gitlab/config:/etc/gitlab \
  --volume /mnt/srv/gitlab/logs:/var/log/gitlab \
  --volume /mnt/srv/gitlab/data:/var/opt/gitlab \

-v 是 --volume 的缩写，代表在docker中，配置、日志、数据，宿主机目录和容器目录的映射关系
latest 指定镜像tag

9443对应容器443端口，即https服务端口，将在最后https部分具体阐述

# 3.6 Web IDE 配置方案说明

Web IDE 功能的存在，让用户可以只操作web浏览器，就能实现文件的维护

  • 最终方案:
  1. docker容器内部端口 443 22 80
  2. docker容器外部端口 9443 9222 80
  • gitlab.rb 配置
external_url 'http://服务器ip/gitlab'
gitlab_rails['gitlab_shell_ssh_port'] = 9222
1
2

external_url: 声明外界http请求的地址，以及WebIDE的地址
gitlab_shell_ssh_port: 声明外界ssh交互的端口

  • 调试流程及结果:
  1. docker容器内部端口80 -> docker外部容器端口9000 -> nginx端口80 => Web IDE 失效
  2. docker容器内部端口9000 -> docker外部容器端口9000 -> nginx端口80 => Web IDE 失效
  3. docker容器内部端口80 -> docker外部容器端口80 -> nginx端口不使用80 => Web IDE 有效

# 5. gitlab权限

# 5.1 用户

# 5.1.1 用户分类

Regular普通用户

可以访问自己所在的群组和项目

Admin管理员

可以访问所有群组、项目、用户，并且能够管理所有功能

其他选项

是否可以创建群组
是否外部用户: 外部用户，只有被明确授权，才能访问内部和私有项目，并且不能创建群组或项目

# 5.1.2 功能

  • 查询用户 http://服务器ip:9000/gitlab/admin/users
  • 创建用户（根据用户分类）http://服务器ip:9000/gitlab/admin/users/new
  • 编辑用户（根据用户分类）http://服务器ip:9000/gitlab/admin/users/用户名/edit
  • 删除用户 http://服务器ip:9000/gitlab/admin/users

# 5.2 角色

# 5.2.1 角色类型

  • Guest 访客
  • Reporter 报告者
  • Developer 开发者
  • Maintainer 维护者
  • Owner 所有者

项目最高权限为Maintainer，群组最高权限为Owner
个人创建项目，一些特殊操作可由项目创建者执行；
群组创建项目，一些特殊操作可以由群组Owner执行。

# 5.3 群组

# 5.3.1 群组描述

群组允许你管理和协作多个项目。组内成员可以访问群组对应的所有项目。
群组可以嵌套创建子群组
属于一个群组的项目以群组的namespace为前缀。
通过项目配置，已存在的项目可以加入群组。
新加入的项目namespace不属于当前组。

# 5.3.2 功能

  • 创建群组 http://服务器ip:9000/gitlab/groups/new
    群组名称 namespace 群组描述 权限

Private 当前群组 和 群组中的项目，只能成员可见
Internal 当前群组 和 所有 internal 项目，可以被所有登录用户访问
Public 不需要任何授权就可以访问 当前群组 和 所有 public 项目

  • 创建项目 http://服务器ip:9000/gitlab/projects/new?namespace_id=Group_Id
    项目名 群组/用户namespace 项目namespace 描述信息

Private群组只可以创建Private项目
Internal群组可以创建Private和Internal项目
Public群组可以创建Private、Internal、Public项目

  • 添加成员 http://服务器ip:9000/gitlab/groups/Group_Namespace/-/group_members
    可以选择成员角色 Guest,Reporter,Developer,Maintainer,Owner (参考5.3角色)，并可选过期时间

# 5.4 项目

# 5.4.1 功能

  • 创建项目 http://服务器ip:9000/gitlab/projects/new
    项目名 群组/用户namespace 项目namespace 描述信息

Private只有被明确授权的用户才能访问项目（添加成员或添加到群组）
